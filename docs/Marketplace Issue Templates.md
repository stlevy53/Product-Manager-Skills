# Marketplace Issue Templates

Use these templates in your own repository to track marketplace distribution work.

## Template 1: Submit to Curated Community List

**Title**
`Submit <repo-name> to <curated-list-name>`

**Body**
```md
## Goal
Add our skill library to <curated-list-name>.

## Why this matters
- Improves credibility and discoverability
- Reaches users actively browsing curated skills

## Proposed listing line
<insert one-line listing text>

## Tasks
- [ ] Fork target repo (if PR-based)
- [ ] Create branch
- [ ] Add listing entry in correct section
- [ ] Open PR with concise rationale
- [ ] Respond to maintainer feedback
- [ ] Merge and record final link

## Definition of done
- PR/listing is live
- Link is recorded in marketplace tracking log
```

## Template 2: Join/Apply to Marketplace

**Title**
`Apply to <marketplace-name> (manual submission)`

**Body**
```md
## Goal
Submit our library to <marketplace-name>.

## Inputs needed
- Name
- Contact email
- Library URL
- Short description

## Tasks
- [ ] Open submission form
- [ ] Enter required details
- [ ] Submit form
- [ ] Capture confirmation (text/screenshot)
- [ ] Update tracking log with status and follow-up date

## Definition of done
- Submission confirmed
- Evidence captured in this issue
```

## Template 3: Improve Marketplace Discoverability Metadata

**Title**
`Update repository metadata for marketplace discoverability`

**Body**
```md
## Goal
Improve findability in marketplace and GitHub search.

## Scope
- Repository topics/tags
- Description text
- Featured skill ordering

## Tasks
- [ ] Update repository topics
- [ ] Refine short description
- [ ] Verify changes are visible publicly
- [ ] Record before/after snapshot in this issue

## Definition of done
- Metadata updates are live and verified
```
